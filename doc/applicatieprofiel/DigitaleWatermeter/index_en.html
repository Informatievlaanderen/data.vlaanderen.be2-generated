This document describes an application profile, in this case OSLO Digital Water Meter (Application Profile). The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The application profile "Digital Water Meter" describes how water consumption in Flanders is monitored with digital meters. These meters are equipped with sensors that record water consumption and provide information about the type, manufacturer, and status of the meter, among other things. The consumption data, consisting of observations of water consumption, provides insight into individual and aggregate consumption, as well as predicted consumption patterns. This profile provides a framework for collecting and sharing this data with third parties.

AggregateConsumptionObservation

Description
Water consumption aggregated across a specific predefined region or group of water meters.
Subclass of
Observation , Measurement
Properties
For this entity the following properties are defined: observedObject , result .
Property Expected Range Cardinality Description Usage Codelist
observedObject Place 1 The predefined zone, in which a group of meters for an observed period of time are located. Zone can be postal code, but also a hamlet or an area demarcated by a polygon.
result QuantitativeValue 1 Total water consumption (volume) measured by or predicted for a group of meters for one period.

IndividualConsumptionObservation

Description
Individual water consumption at one connection point related to one meter.
Subclass of
Observation , Measurement
Properties
For this entity the following properties are defined: observedObject , result .
Property Expected Range Cardinality Description Usage Codelist
observedObject Junction 1 The connection point linked to the consumption measured by one water meter. This can be a detached house, or one apartment in a larger building. There can be multiple connection points on one connection (e.g. in apartment building)
result QuantitativeValue 1 The water consumption measured or predicted at one connection point related to one time (index) or period (volume). Individual means that it is the measurement of one meter. Index = position of water counter at time of phenomenon time. Volume = volume consumed in the period specified as phenomenon time.

Datatypes

Address Presentation

Description
More readable representation with only the basic data of the address, intended for the use of an address as an attribute of another object.
Properties
We have defined the following properties for this datatype: Address area , Unit Level 1 , Unit Level 2 , Bus number , land , Full Address , house number , Location indication , Location name , Municipality name , mailbox , Zip code , Postal Name , Refers , street name .
Property Expects Type Cardinality Description Usage Codelist
Address area LangString 0..1 Name of geographical area or locality grouping together a number of addressable objects in order to address them without the area or locality being an administrative unit E.g. the name of the hamlet in which the address is located.
Unit Level 1 LangString 0..1 Highest Administrative Unit of the address, usually a country.
Unit Level 2 LangString 0..1 The region of the address, typically a county or state or similar area that typically includes multiple cities.
Bus number String 0..1 Officially assigned alphanumeric code that is added to the house number to distinguish several building units, pitches, berths or plots that have the same house number. Specialization of Address Representation.Location Indication for Belgian Addresses.
land LangString 0..1 Country in which the address is located.
Full Address LangString 0..1 The complete address in one string, formatted or not. Avoids errors due to splitting the address into its parts. Displays the prescribed order of the different parts.
house number String 0..1 Alphanumeric code officially assigned to building units, berths, pitches or plots. Specialization of Address Representation: location indication for Belgian addresses.
Location indication String 0..* Alphanumeric code that uniquely identifies the address locator within the street, administrative unit, etc.
Location name LangString 0..* Name or description of the geographical object that the address locator indicates.
Municipality name LangString 0..1 Municipality name of the address.
mailbox String 0..1 P.O. Box number in a post office.
Zip code String 0..1 Code indicating the geographical area that groups addresses for postal purposes.
Postal Name LangString 0..1 Name indicating the geographical area grouping addresses for postal purposes.
Refers Belgian Address 0..1 Address from which the address representation is derived. This is only possible for Belgian addresses, as address is understood to be a Belgian address.
street name LangString 0..1 Street name of the address.
