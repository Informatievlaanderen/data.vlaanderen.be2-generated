Ce document décrit un profil d’application, dans ce cas Associations de fait - Profil d’application. Ce profil d’application répond à la question de savoir comment le modèle de domaine correspondant peut être appliqué dans la pratique. De plus, les limitations (cardinalité, listes de codes) sont expliquées et les termes correspondants (RDF) sont listés.

Résumé

Ce profil d’application vous permet de décrire ActualAssociations.

Une association d’établissement des faits est une organisation qui, comme les autres associations, a un but désintéressé. La différence est qu’une association de fait n’est pas enregistrée en tant qu’entité juridique, elle n’est qu’une association parce que les membres entreprennent des activités ensemble.

Les associations réelles peuvent être reconnues, ce qui leur donne un caractère formel sans être une entité juridique.

Afin de pouvoir utiliser les services publics, par exemple le prêt d’équipements sportifs par une administration locale, il peut suffire de fournir quelques informations de base, mais parfois cette reconnaissance est également nécessaire.

Le traitement de la demande de reconnaissance et de la reconnaissance finale est effectué par un organisme public. Afin d’obtenir la Reconnaissance, certaines preuves doivent être fournies par l’Association Actuelle sur la base de critères spécifiques.

Le profil de candidature permet également de décrire qui est membre de la Fact-Association, et/ou qui est responsable (et quelle est cette responsabilité).

Enfin, il est possible de décrire quels services publics ont été sollicités par la Fact-Association.

Le profil d’application réutilise les termes et la manière dont ils y sont appliqués à partir du profil d’application OSLO Organization et du catalogue de services OSLO.

Une description plus formelle des termes peut être trouvée dans le vocabulaire OSLO-FactualSocieties.

Demande de reconnaissance

Description
La demande de désignation par une institution compétente établissant qu’une personne ou une institution remplit certaines conditions, qu’elles soient temporaires ou non.
Sous-classe de
Services publics (consommation)
Propriétés
Les propriétés suivantes sont définies pour cette entité: aangevraagdBij , activité , Critère requis , Décision de reconnaissance , description , entrée , hasResult , période de validitéReconnaissance , type .
Propriété Type attendu Cardinalité Description Utilisation Liste de codes
aangevraagdBij Organisme public 1..*
activité OrganisatieActiviteit 0..* Type d’activité exercée par un organisme.
Critère requis Critère requis 0..* Référence à l’exigence du critère qui s’applique à la demande de reconnaissance.
Décision de reconnaissance String 0..1 Le résultat de la demande de reconnaissance.
description ChaîneLangue 0..1 Une brève description de la demande de reconnaissance.
entrée Preuve 0..* Document(s) justificatif(s) qui doivent être soumis afin d’obtenir la reconnaissance.
hasResult Reconnaissance 0..1 La reconnaissance qui est accordée sur la base de la demande de reconnaissance soumise.
période de validitéReconnaissance Période 0..1 Intervalle de temps, indiqué dans la demande de reconnaissance, pour lequel l’agrément demandé s’appliquera.
