This document describes an application profile, in this case De facto associations - Application profile. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

This application profile allows you to describe ActualAssociations.

A Fact-Finding Association is an Organization that, like other Associations, serves a disinterested purpose. The difference is that a De facto Association is not registered as a legal entity, it is merely an Association because the members undertake activities together.

Actual associations can be recognized, which gives them a formal character without being a legal entity.

In order to be able to make use of Public Services, e.g. the lending of sports equipment from a local government, it may be sufficient to provide some basic information, but sometimes this Recognition is also necessary.

The processing of the Application for Recognition and the final recognition is done by a Public Organisation. In order to obtain the Recognition, certain proofs must be provided by the Actual Association on the basis of specific criteria.

The application profile also makes it possible to describe who is a member of the Fact-Association, and/or who is responsible (and what that responsibility is).

Finally, it is possible to describe which Public Services have been requested by the Fact-Association.

The application profile reuses terms and the way in which they are applied there from the OSLO Organization application profile and OSLO Services Catalog.

A more formal description of the terms can be found in the vocabulary OSLO-FactualSocieties.

Entities

Application for recognition

Description
The request for designation by a competent institution establishing that a person or institution meets certain requirements, whether temporary or not.
Subclass of
Public services (consumption)
Properties
For this entity the following properties are defined: aangevraagdBij , activity , Criterion requirement , description , hasResult , input , Recognition decision , type , validity periodAcknowledgment .
Property Expected Range Cardinality Description Usage Codelist
aangevraagdBij PublicOrganization 1..*
activity OrganisatieActiviteit 0..* Type of activity carried out by an organization.
Criterion requirement Criterion requirement 0..* Reference to the criterion requirement that applies to the application for recognition.
description LanguageString 0..1 A brief description of the application for recognition.
hasResult Recognition 0..1 The recognition that is granted on the basis of the submitted application for recognition.
input Proof 0..* Supporting document(s) that must be submitted in order to obtain recognition.
Recognition decision String 0..1 The result of the application for recognition.
type Accreditation type 1 The nature of the recognition. Recognition type can be described by a name or a code. The name is then e.g. recognised as 'sports club', and the code is e.g. '20'. This code is a fictitious code, which may have meaning in the public organization that issues the recognition.
validity periodAcknowledgment Period 0..1 Time interval, stated in the application for recognition, for which the approval that has been applied for will apply. The time interval to which the validity period of an Application for Recognition is entitled may differ from the Recognition that is finally granted.

Association

Subclass of
Organization
Properties
No properties have been defined for this entity.

FactualAssociation

Description
An Association without legal personality, based on an agreement between two or more persons and/or organisations to pursue a selfless purpose within the framework of one or more well-defined activities.
Subclass of
Association
Properties
For this entity the following properties are defined: asksTo , Bank account , hasAcknowledgment , hasEstablishment , hasMember , hasRepresentative , hasSub-organisation , Identifier , isGelinktAan , isSub-organizationOf , activity , Contact Info , Preferred name , Submits .
Property Expected Range Cardinality Description Usage Codelist
asksTo Public Service 0..* Reference to the public service requested by the de facto association.
Bank account TypedString 0..1 A unique number assigned to an account at a bank.
hasAcknowledgment Recognition 0..* Reference to the recognition(s) that the de facto association has.
hasEstablishment Establishment 0..* An office or other building where an organization is located. Organizations can be spread across multiple sites, and branches can strike in multiple locations.
hasMember Agent 2..* Reference to the members who are part of the de facto association.
hasRepresentative Representative 0..* Reference to the representative of the de facto association.
hasSub-organisation FactualAssociation 0..* The de facto association has a sub-organization.
Identifier Identifier 1 Identifier of the object.
isGelinktAan RegisteredOrganisation 0..*
isSub-organizationOf FactualAssociation 0..* The de facto association is a sub-organization of another de facto association.
activity OrganisatieActiviteit 0..* Type of activity carried out by the organization. Specialization of the classification attribute (in addition to type & status).
Contact Info Contact Info 0..* Information such as email, telephone... which allows them to contact the Organisation.
Preferred name LanguageString 1 Name with which the organization is preferably designated.
Submits Application for recognition 0..* The application for recognition submitted by the de facto association.

house number String 0..1 Alphanumeric code officially assigned to building units, berths, pitches or plots.
land LanguageString 0..1 Country in which the address is located.
