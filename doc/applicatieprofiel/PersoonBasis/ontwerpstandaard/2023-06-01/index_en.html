This document describes anapplication profile, in this case Person. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The OSLO-Person application profile merges terms from vocabularies about persons and mainly from OSLO-Person vocabulary.

.

In principle, an application profile is intended for a certain application. Together with the aim of the application, additional constraints such a cardinalities and prescribed code lists are present. Here, this is not the case and therefore we speak of a basic profile. It contains the info that is seen as relevant for OSLO for the exchange of information about persons. Nevertheless, the profile pools terms that are predominantly used in the context of a civil register.

The profile contains terms that are related to the identification of a person (name, gender, birth day) but also terms that described the civil characteristics of people (personal relations such as marriage, family and relations with the state such as state citizenship and residency/domicile). We refer to the OSLO-Person vocabulary for more details.

The profile imposes minimal limitations, only the information identifying the person are mandatory. Currently there are no prescribed code lists.

Conformiteit

An application profile is a specification for the exchange of information that introduces additional limitations for the application of vocabularies. Such additional limitations can contain the following elements:

  • refinement of the terminology (classes and properties) consistent with the semantics from the concerning specifications with a specific use as aim;
  • external terminology (classes and properties) used for new/extra terms that do not apprear in existing vocabularies.

In order to be in accordance with this applicationprofile, the implementation:

  • MUST always contain the properties that have cardinality 1 for every class.
  • CAN NOT contain more than 1 instance of properties that have 1 as maximum cardinality.
  • CAN use terminology (classes and properties) in a way that is consistent with the semantics (definition, use, domain, range).
  • CAN NOT use terminology of other controlled vocabulary than the one which is defined in this applicationprofile.
  • CAN be expanded with classes and properties of other data models (vocabulary) that do not overlap with the terminilogy from this applicationprofile.
