This document describes an application profile, in this case Soil (Application Profile). The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The Soil application profile explains how the terminology in this domain should be used to exchange data on soil observations.

The application profile builds on the OSLO Sensors and Sampling application profile, among other things.

This subdomain relates to the sampling and observation of the soil.

Sampling is done at Soil Locations (sometimes further specialized as Soil Borehole or Profile Well).

In addition to analyses of soil samples, the soil profile is determined and the soil as a whole is classified according to various soil classification systems.

The Profile of a Soil as a result of an Observation consists of Horizons and Layers that can also be sampled or observed separately.

Also in this domain there are Observations that can be described generically.
