This document describes an application profile, in this case Soil drilling (application profile). The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The Soil Drilling application profile explains how the terminology in this domain should be used to exchange data on drilling in the ground or more generally sampling the soil.

The application profile builds on the OSLO Sensors and Sampling application profile, among other things.

This subdomain relates to the activity of Drilling in Soil & Subsoil. Its purpose is typically to sample Soil & Subsoil vertically with a view to determining changes in depth in terms of stratification, lithology, etc.

This is done by analyzing soil samples or performing observations at different depths.

The resulting borehole also allows devices to be hosted for more permanent monitoring or for exploitation.

Drilling is a specialised activity that usually takes place within a Legal Framework and with special equipment and personnel.

Sampling object , Sample

Property Expected Range Cardinality Description Usage Codelist
CurrentLocation Location 0..1 Place where the Sample is stored. This is typically an indirect indication of the location, e.g. a reference to the Organisation where the Sample is kept and possibly its position within it (shelf in a storage or drawer in a museum).
Material class Material class 1 Basic classification of the material that makes up the Soil Sample. E.g. soil, rocks...
Sample Type Soil sample type 0..* Material form that the Soil Sample takes. E.g. whether it is rinsed, stung, cored... is.
Sampling site Geometry 0..1 Place where the Sample was taken. NOTE

size Measure 0..1 The physical size of the sample. Depending on the type of material, this is about length, mass, volume, etc.
