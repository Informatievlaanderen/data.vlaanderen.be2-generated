This document describes an application profile, in this case Groundwater monitoring network (application profile). The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The Groundwater Monitoring Network application profile explains how terminology in this domain should be used to exchange data on groundwater observations.

The application profile builds on the OSLO Sensors and Sampling application profile, among other things.

This subdomain relates to the monitoring of groundwater level and quality.

The level is measured in so-called monitoring wells, usually with open or closed piezometers placed in these wells.

Monitoring can be done manually or automatically with Sensors.

The quality is determined by observing various quality parameters on Groundwater samples obtained in the Wells and the Piezometers.

The piezometers are organized in so-called Monitoring Networks.
