This document describes an application profile, in this case Soil and subsoil (application profile). The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The Soil and Subsoil application profile explains how to use the terminology that is additionally provided to exchange Observations specifically for the Soil and Subsurface domain (but still generic enough to be useful for other subdomains of Soil and Subsoil).

This concerns, for example, BO_RuimtelijkBemonsteringsobject that adds attributes such as ground level and depth to Spatial Sampling Object from Observations and Measurements .

Also for the domain Soil and Subsurface, the Spatial Sampling Object Profile from Observations and Measurements was expanded with Profile Elements with the intention of better describing parts of Profiles such as Layers.

In addition to Ranges that can document variations of phenomena in space and time, there was also a need for "ordinary" measurement series in the domain of Soil and Subsurface, where the phenomenon can be a function of an arbitrary parameter. For that reason, we added Measurement Series and the widely used Fraction Measurement specialization.

Entities

BO_RuimtelijkBemonsteringsobject

Description
Spatial Sampling Object with additional characteristics relevant to the Soil and Subsoil domain.
Subclass of
SpatialSampling Object
Properties
For this entity the following properties are defined: depth , Ground level , location .
Property Expected Range Cardinality Description Usage Codelist
depth Depth 0..1 The depth of the sampling object. This is important if the Sampling Object is, for example, a borehole and it has not been drilled perfectly vertically so that depth and length differ from each other.
Ground level Ground level 0..* The ground level of the sampling object. The exact location of the ground surface is very important, e.g. in the case of a borehole, because other data are compared to it (e.g. the depth of the drilling).
location Location 0..* The location of the sampling object. In addition to the geometry, additional indications of the location are sometimes useful, e.g. in the case of the borehole in which municipality the borehole has been drilled.
