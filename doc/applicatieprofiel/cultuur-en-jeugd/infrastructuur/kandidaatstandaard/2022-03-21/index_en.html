This document describes an application profile, in this case Application profile Culture and Youth Infrastructure. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

This application profile makes it possible to describe cultural and youth infrastructure and can be used to exchange infrastructure data.

As a central class, there is the Infrastructure that can be represented in a hierarchy by means of sub/super relationships. This Infrastructure can further specialize in matters such as a Building, Indoor or Outdoor Space.

An Activity or Thing may be linked to this Infrastructure.

Matters such as Ownership, Management, Use and Reservation are also described.

It also describes, among other things, what the Accessibility is, what the Accessibility is, etc.

The application profile reuses terms and the way they are applied from the application profile OSLO-Organization, OSLO-CulturalHeritage:Object, OSLO Person, OSLO-Consent, OSLO Association, Actual OSLO-Purpose-Driven Digital Transformation, OSLO Service, OSLO observations and measurements, OSLO Building and OSLO Plot.

A more formal description of the terms can be found in the vocabulary OSLO-Culture & Youth Infrastructure.
