This document describes an application profile, in this case OSLO Data Quality. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The application profile shows how the terms from the OSLO vocabulary Data Quality (which is based on ISO19157:2013) should be used to describe data quality.

This application profile makes it possible to describe the quality of data, i.e. the extent to which data is potentially suitable for a certain use.

We only retained what is necessary to describe aspects of quality at the smallest granular level, e.g. the positional accuracy of a location but not that of an entire geographic dataset.

Aspects such as purpose, use and origin of data fall under metadata and are not considered here.

The profile revolves around the Quality Element class, of which the authorities describe a certain quality aspect by means of a certain measure obtained according to a certain method.

The aspects are classified as subclasses of Quality Element and include things like accuracy, completeness, and consistency. For subclasses of these, please refer to ISO1957 and the corresponding URIs in isotc211:DataQuality.

An individual quality aspect is considered to be an observation with a quantitative result or a declaration of conformity or a description of quality.

It may include an indication of the method that led to that result and a reference to the measure used to express the quality.
