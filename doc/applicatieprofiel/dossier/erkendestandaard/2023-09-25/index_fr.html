Ce document décrit un profil d’application, dans ce cas Dossier OSLO - Profil de l’application. Ce profil d’application répond à la question de savoir comment le modèle de domaine correspondant peut être appliqué dans la pratique. De plus, les limitations (cardinalité, listes de codes) sont expliquées et les termes correspondants (RDF) sont listés.

Résumé

Il s’agit d’un profil d’application de base dans le vocabulaire du dossier OSLO.

Le modèle d’information conceptuel constitue la base de l’échange de données de dossier entre les différents services. Il définit un certain nombre d’entités abstraites, de propriétés pertinentes et de leurs interrelations. Ce modèle d’information conceptuel constitue la base que les développeurs et les gestionnaires de bases de données peuvent développer en un modèle de données physique. Le vocabulaire donne un aperçu de la terminologie utilisée et des définitions associées, le profil d’application de base rassemble ces termes dans une spécification générique pour l’échange de données.

À titre d’illustration, des exemples de cas d’utilisation sont élaborés dans ce document.
