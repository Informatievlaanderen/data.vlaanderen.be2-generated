Ce document décrit un profil d’application, dans ce cas Infrastructure cyclable d’OSLO (profil d’application). Ce profil d’application répond à la question de savoir comment le modèle de domaine correspondant peut être appliqué dans la pratique. De plus, les limitations (cardinalité, listes de codes) sont expliquées et les termes correspondants (RDF) sont listés.

Résumé

Le profil d’application OSLO Cycling Infrastructure explique comment la terminologie de ce domaine doit être utilisée pour échanger des données pertinentes sur les infrastructures cyclables.

Le profil d’application est composé de trois parties :

  • L’infrastructure (physique),
  • le réseau (logique), et
  • les propriétés.

En premier lieu, il est fait référence à la mise en œuvre physique de l’infrastructure à l’aide de la norme OSLO Public Domain, dans lequel le concept de Élément d’infrastructure. Pour ce profil d’application, un objet Bicycle Infrastructure Object plus spécifique a été créé, afin de classer ces éléments d’infrastructure en fonction de Bicycle Infrastructure également. Les éléments d’infrastructure peuvent être liés aux éléments du réseau cyclable, qui constituent la partie logique du modèle.

Les éléments du réseau cyclable sont les éléments constitutifs du profil d’application et peuvent être des nœuds, des segments, des séquences, etc. qui forment ensemble un réseau cyclable. La partie réseau du profil d’application est basée sur INSPIRE (Inspire Transport Networks) et s’appuie sur Réseau OSLO et OSLO-Transport. Les éléments du réseau cyclable comprennent le Présentation logique de toutes les installations cyclables telles que les routes, les pistes cyclables, les chemins de halage, les pistes cyclables, etc. sur lequel le cycliste peut faire du vélo.

De plus, on parle des propriétés qui sont présentes ou applicables au réseau, comme la largeur de la piste cyclable, l’éclairage de la route, dans quel sens vous êtes autorisé à faire du vélo. Ceci est inclus dans le modèle en tant que propriété Transport. Par Propriétés de Transport générales, nous entendons les propriétés qui peuvent s’appliquer à Plusieurs réseaux de transport (par exemple, l’éclairage routier ne s’applique pas seulement aux vélos, mais aussi aux voitures). De plus, nous définissons plus précisément les propriétés du réseau cyclable, où nous voulons indiquer qu’il s’agit de spécifiquement sur l’échange d’informations liées au vélo, par exemple la largeur du vélo, DirectionCirculation cyclable et chaussée cyclable.

La relation entre un élément de réseau cyclable et une entité de transport est créée en tant qu’élément de réseau cyclable hérité d’un objet de transport, auquel une propriété de transport peut être liée. Le lien n’est pas explicitement présent dans le modèle, mais pour cela référence est faite à Inspire.

Pour indiquer qu’un bien de transport ne se trouve que sur une partie limitée d’un L’élément de réseau cyclable est valide, une référence linéaire est utilisée.

Présentation de l’adresse

Description
Représentation plus lisible avec seulement les données de base de l’adresse, destinée à l’utilisation d’une adresse comme attribut d’un autre objet.
Propriétés
Les propriétés suivantes sont définies pour ce type de données: Adresse complète , boîte aux lettres , Code postal , Nom de la commune , Nom de la rue , Numéro de bus , Numéro de maison , terre .
Propriété Type attendu Cardinalité Description Utilisation Liste de codes
Adresse complète Chaîne de langue 0..1 L’adresse complète dans une chaîne, formatée ou non. Évite les erreurs dues à la division de l’adresse en ses parties. Affiche l’ordre prescrit des différentes pièces.
boîte aux lettres String 0..1 Numéro de case postale dans un bureau de poste.
Code postal String 0..1 Code indiquant la zone géographique qui regroupe les adresses à des fins postales.
Nom de la commune Chaîne de langue 0..1 Nom de l’adresse de la municipalité.
Nom de la rue Chaîne de langue 0..1 Nom de la rue de l’adresse.
Numéro de bus String 0..1 Code alphanumérique officiellement attribué qui est ajouté au numéro de maison pour distinguer plusieurs unités de construction, emplacements, couchettes ou parcelles qui ont le même numéro de maison. Spécialisation de la représentation d’adresses : indication de la localisation des adresses belges.
Numéro de maison String 0..1 Code alphanumérique officiellement attribué aux unités de construction, aux postes d’amarrage, aux emplacements ou aux terrains. Spécialisation de la représentation d’adresses : indication de la localisation des adresses belges.
terre Chaîne de langue 0..1 Pays dans lequel se trouve l’adresse.
