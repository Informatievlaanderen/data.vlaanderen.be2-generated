This document describes an application profile, in this case Financial Reporting Application Profile. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

This application profile was developed to semantically model the different data streams related to financial reporting and to standardize the structure of such data.

Many organisations have to submit their financial reports (annually) to various (public) organisations in order to check or support a dossier for subsidies, among other things.

The following information is captured with regard to the financial report: the financial facts stated in it, the financial concept behind the fact (e.g. assets, turnover, costs, usually based on an imposed schedule), the context in which the fact must be viewed (e.g. the period to which the fact relates), if applicable. the dimensions that apply (e.g. time, region, product).

The model for the financial report is based on a well-known standard in this domain, i.e. XBRL , and a sub-standard thereof, i.e. XBRL Dimensions.

As far as the data flows are concerned, a link is made to the filing of a financial report and to the various validation steps it goes through, e.g. for approval of a grant. A financial report can also be linked to a file.

A more formal description of the terms used in the application profile can be found in the OSLO – Financial Reporting vocabulary.

Entities

Concept

Description
Definition of a type of Fact.
Properties
For this entity the following properties are defined: abstract , Associated With , Balance Type , Datatype , definition , Hypercube , id , Item Type , label , name , periodType .
Property Expected Range Cardinality Description Usage Codelist
abstract Boolean 1 Indicates whether a Fact of this type may be instantiated. This is not the case, for example, for a Concept that is the parent of several other Concepts, e.g. Operating income that is split into Turnover, Orders, Inventory, etc.
Associated With Concept 0..* Association between two Concepts. E.g. Revenue is the parent-Concept of Revenue.
Balance Type Balance Type 0..1 Indicates whether a Fact of this type is typically credited or debit to a balance sheet. For Monetary Facts.
Datatype URI 1 The data type of a Fact of this type. E.g. Monetary, Equities.
definition Language String 1 Definition of the Concept.
Hypercube Hypercube 0..* Hypercube with which Facts of this type are associated. A Hypercube is a set of Dimensions that can apply to a Fact, e.g. Revenues are broken down by Fiscal Year, Product and Region.
id Identifier 1 Concept Identifier.
Item Type Item Type 1 Indicates whether the data type of this Fact type is primitive or complex. NOTE
label Language String 0..* Keyword assigned to the Concept according to some classification system.
name Language String 1 The name of the Concept.
periodType PeriodType 0..1 The manner in which the period to which this type of Fact relates is expressed. For example, the period in which the Revenue was realised is expressed as a moment (e.g. 2020) or as a period (e.g. 1/1/2020 to 31/12/2020).

Concept Relationship

Description
Relationship between two Concepts.
Properties
For this entity the following properties are defined: Concept (source) , Concept (target) , order , role .
Property Expected Range Cardinality Description Usage Codelist
Concept (source) Concept 1
Concept (target) Concept 1
order Integer 0..1 Arrangement of the Concept in a certain order. E.g. the children of the Revenue Concept are arranged in the Financial Report as follows: Turnover at number 1, Orders at number 2, Inventory at number 3 etc.
role Concept Relationship 1 Nature of the association between the Concepts.

Context

Description
Framework in which a Fact is to be interpreted.
Properties
For this entity the following properties are defined: dimension , entity , period , report , scenario .
Property Expected Range Cardinality Description Usage Codelist
dimension Dimension 0..* Perspective from which the Fact has been looked at. E.g. for which period the Fact is valid or to which product it refers or to which region or all these Dimensions together. E.g. Turnover123 applies to the current financial year and to product ABC (Dimensions are therefore Period and Product with respectively value "current financial year" and "product ABC"). For example, turnover 456 in the same report may apply to the same financial year but to product DEF.
entity Organization 1 Organization to which the Fact relates. In the case of the annual accounts, it is about the company whose annual accounts are filed
period TemporalEntity 1 The moment or period to which the Event relates.
report FinancialReport 1 Financial Report to which the Context belongs. To indicate to which report the Context belongs without having to consult the individual Facts. After all, the same Context can be reused for multiple Facts or even for all Facts in a given report.
scenario String 0..1 Additional Information About the Fact.

Dimension

Description
Independent characteristic by which a Fact is characterized.
Properties
For this entity the following properties are defined: concept , value .
Property Expected Range Cardinality Description Usage Codelist
concept Dimension concept 1 Reference to the dimension concept. E.g. to indicate that it is a Dimension of the type "Period". NOTE: Dimension concept can be understood as a metaclass, so this can also be implemented by means of subclasses of Dimension: if the subclass is e.g. Period, then the reference to the Dimension concept is no longer necessary.
value AnyDatatype 1 Value that the Dimension assumes. E.g. "current financial year" for a Dimension of type Period.

Dimension concept

Description
Definition of a Dimension type.
Properties
For this entity the following properties are defined: abstract , Associated With , Datatype , definition , id , Item Type , label , name .
Property Expected Range Cardinality Description Usage Codelist
abstract Boolean 1 Indicates whether a Dimension of this type may be instantiated. This is not the case, for example, for a Dimension concept that is the parent of several other Dimension concepts, e.g. if region is split into Region Size and Region Type.
Associated With Dimension concept 0..* Association between two Dimension concepts. Egb Region is the parent of Region Size.
Datatype URI 1 The data type of a Dimension of this type. E.g. Product Type Code List for Product.
definition Language String 1 Definition of the Dimension Concept.
id Identifier 1 Dimension Concept Identifier.
Item Type DimensieItemtype 1 Specifies whether the data type of this Dimension type is primitive or complex. If the value of a Dimension consists of multiple parts, it is complex, e.g. Product consisting of a product category and a product name.
label Language String 0..* Keyword assigned to the Dimension concept according to some classification system.
name Language String 1 the name of the Dimension concept.

Fact

Description
An individual piece of information in a Financial Report.
Properties
For this entity the following properties are defined: concept , context , report , value .
Property Expected Range Cardinality Description Usage Codelist
concept Concept 1 Reference to the Concept that is behind the fact. E.g. to indicate that it concerns a Fact of the type "Turnover". NOTE: Concept can be understood as a metaclass, so this can also be implemented by means of subclasses of Fact: if the subclass is e.g. Revenue, then the reference to the Concept is no longer necessary.
context Context 1 Reference to the Context in which the Fact is to be viewed. With, for example, the Period to which the Fact relates.
report FinancialReport 1 Reference to the financial report of which the fact is a part.
value AnyDatatype 1 The Value of the Fact. E.g. for a Fact of the type Turnover: the amount of the Turnover.

FinancialReport

Description
Report on the financial condition or economic position of an Organization.
Subclass of
ZakelijkRapport
Properties
For this entity the following properties are defined: aboutAbout , Associated With , Contains , context , file , Filing , id , language , name , period , scheme , type , validationStatus .
Property Expected Range Cardinality Description Usage Codelist
aboutAbout Organization 1 A financial report is always about a specific organization
Associated With FinancialReport 0..* Other FinancialReport with which the report is associated.
Contains Fact 0..* Fact that is stated in the report.
context Context 0..* Reference to a Context in which the Facts in the report are viewed.
file File 0..* File relating to the report.
Filing Filing 0..* Refers to Report Filing. More specifically, when and to which Organisation the report was filed.
id Identifier 0..1 Identifier of the report.
language GetypeerdeString 1 The language in which the report was written. NOTE: Datatype is a Typed String so that the language can easily be represented by means of a String, but with the type of String indicated so that it can be interpreted correctly. E.g. value "nl" and type "iso 639-1".

period TemporalEntity 1 Time or period to which the report relates. Typically, the duration follows from the specified start and end time, but can also be a moment here.
scheme Scheme 0..1 Schedule that the report complies with.
type Chose 1 Nature of the report. E.g. financial statements, balance sheet, income statement, etc.
validationStatus Validation 0..* Reference to the status of the Report for the purpose of its Validation.
