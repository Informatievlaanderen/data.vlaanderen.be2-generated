This document describes an application profile, in this case Application Profile Learning Enrolment Credential. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

This application profile reuses the vocabulary OSLO Learning Credential, which in turn is based on the European Learning Model. ELM is based on the W3C Verifiable Credentials Data Model in terms of credential exchange and validation.

This application profile focuses on the collection and exchange of enrolment data in an educational context. To this end, a number of extensions to ELM are planned (registration, etc.). An explicit Learning Enrollment Credential is created to cover these use cases.

Entities

Learning Credential

Description
A set of one or more claims about learning experiences made by a publisher.
Subclass of
Verifiable Credential
Properties
For this entity the following properties are defined: appendix , description , display , expiry date , Identifier , Includes , owner , publisher , title , type , valid from .
Property Expected Range Cardinality Description Usage Codelist
appendix Media Object 0..* Any digital document (PDF, JPEG or PNG format) that an issuer has attached to the European Digital Credential document.
description Note 0..1 Description of the learning credential.
display Display Parameter 1 The presentation of the Learning Credential.
expiry date DateTime 1 The expiration date.
Identifier Identifier 0..* An alternative identifier of the European Digital Credential for Learning typically assigned to the credential by the issuing organisation.
Includes Learning Credential 0..* A Learning Credential that includes another Learning Credential.
owner Person 1 The person (subject) about which claims are made and who owns the credential. This property is inherited from Verifiable Credential. This trait is inherited from verifiable credentials.
publisher Organization 1 The organisation that issued the credential and sealed it with its digital e-seal. This property is inherited from Verifiable Credential.
title Language String 1 Textual description of the learning credential.
type Credential Type 1..* The type of the European Digital Credential. It should be provided using the EDC Controlled List of Credential Types. It should be provided using the EDC-checked list of credential types. Link
valid from DateTime 1 The earliest date on which the learning credential becomes valid.

Verifiable Credential

Description
A credential is a set of one or more claims made by a publisher. A verifiable credential is a tamper-proof credential that has authorship that can be cryptographically verified.
Properties
No properties have been defined for this entity.

The credit points.
