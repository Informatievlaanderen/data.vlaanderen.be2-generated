This document describes an application profile, in this case Local Economy - Application Profile. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The Local Economy application profile focuses on collecting, analyzing, and managing data related to footfall-by, sales transactions, and factors that may affect the local economy.

This application profile was developed to semantically model the different data streams related to local economy data and to standardize the structure of such data. It focuses on collecting, analyzing, and managing data related to footfall-by, sales transactions, and all factors that may affect the local economy.

The following information is captured with regard to data relating to the local economy: the transactions that take place at organisations, passers-by data, traffic data, environmental observations (e.g. weather, events, etc.), organised events, etc.

A more formal description of the terms used in the application profile can be found in the vocabulary OSLO – Local Economy.

Address Presentation

Description
More readable representation with only the basic data of the address, intended for the use of an address as an attribute of another object.
Properties
We have defined the following properties for this datatype: Bus number , house number , land , Municipality name , Refers , street name , Zip code .
Property Expects Type Cardinality Description Usage Codelist
Bus number String 0..1 Officially assigned alphanumeric code that is added to the house number to distinguish several building units, pitches, berths or plots that have the same house number. Specialization of Address Representation: location indication for Belgian addresses.
house number String 0..1 Alphanumeric code officially assigned to building units, berths, pitches or plots. Specialization of Address Representation: location indication for Belgian addresses.
land Language String 0..1 Country in which the address is located.
Municipality name Language String 0..1 Municipality name of address.
Refers Belgian Address 0..1 Address from which the address representation is derived. This is only possible for Belgian addresses, as address is understood to be a Belgian address.
street name Language String 0..1 Street name of the address.
Zip code String 0..1 Code indicating the geographical area that groups addresses for postal purposes.

Timetable

Properties
We have defined the following properties for this datatype: End , Exception date , expensive , Finish , frequency , numberRepeats , perDay , perDayOfTheMonth , perMonth , perSetposition , Start time , Starting date .
Property Expects Type Cardinality Description Usage Codelist

perMonth Integer 0..* Months of the year when it takes place.
perSetposition Integer 0..1 Specification of the day(s) within a period on which it takes place. NOTE
Start time Time 0..1 Time when it starts.
Starting date Date 0..1 Date from when it takes place.
