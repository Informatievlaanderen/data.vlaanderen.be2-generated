This document describes an application profile, in this case Application profile Mobility budget. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The objective of this application profile is to standardise the flow of information about the mobility budget (for citizens). This application profile was developed as part of the CoT project of the cities of Hasselt, Leuven and Antwerp to arrive at a Mobility Budget for citizens.

The application profile reuses terms and the way they are applied from the following standards, among others:

  • Application Profile OSLO Generic
  • Application Profile OSLO Person Basic
  • Application profile OSLO Mobility: trips and offers
  • Application Profile OSLO Organization Basic

To reduce the traffic burden in the busy city centres, the mobility budget has been created. This budget for citizens encourages them to take more sustainable alternatives than the traditional car. This budget can be allocated by a local government, another government body, local merchants and mobility services themselves. All of them can send a budget to certain citizens in the form of cash discounts, free rides or other promotions. Users can then purchase a ticket for themselves or other people to get around. There are also conditions for this budget, budgets can be limited per age group, time aspect, location,...

Terms related to the Mobility Budget in the broadest sense have been included in the Mobility Budget Vocabulary.

Entities

Condition

Description
Condition attached to the budget
Properties
For this entity the following properties are defined: description , Itinerary , Mobility budget , Mobility service , number , period , segment , Service Provider Type , specification .
Property Expected Range Cardinality Description Usage Codelist
description Language String 0..1 Description of the condition
Itinerary Location 0..* Sequence of Locations to be visited during the Trip.
Mobility budget Mobility budget 0..* Mobility budget to which the Condition applies.
Mobility service Mobility service 0..* Services that enable movement of Individuals between Locations. Only these services may be used with the mobility budget.
number Integer 0..* Number of journeys that can be made with the Ticket. Certain discounts are only valid a certain number of times. This value indicates how many times it can be used
period TemporalEntity 0..* Period during which the mobility budget is valid
segment Segment 0..* Types of segment to which an agent belongs The segments that are allowed to use the mobility budget
Service Provider Type Service type 0..* Nature of the service provider type The mobility budget may only be used for 1 or more service providers. Link
specification Document 1 Document in which additional specifications are described Examples are: discount mechanisms, etc.
