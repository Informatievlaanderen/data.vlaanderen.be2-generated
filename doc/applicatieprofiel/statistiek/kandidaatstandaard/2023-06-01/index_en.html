This document describes an application profile, in this case OSLO Statistics. The application profile specifies a semantic data model covering a set of usecases. The specification consists of terms with their constraints (cardinalities, codelists, usage guidelines).

Summary

The OSLO Statistics application profile shows how concepts from the vocabulary of the same name should be used to describe a Statistical Observation.

By a Statistical Observation we mean an Observation whose result is a Statistic. This is useful to be able to distinguish between e.g. an individual measurement or a measurement that is the average of a number of individual measurements.

For more information about the Observation class, please refer to the OSLO Observations & Measurements application profile (further supplemented in the OSLO Sensors & Sampling) application profile.

Statistics as a discipline is concerned with gaining insight into a certain phenomenon by means of numbers.

The collected data is reduced to what we call a Statistic, rather than the individual figure we mean information that describes a certain characteristic of the dataset as a whole or of a variable from the dataset.

We based ourselves on the UNCERT model to describe this class.

Typically, these are summary measures such as mean, median, total, or an index. Or it concerns measures that say something about the distribution of the figures, e.g. minimum/maximum, standard deviation and the like. This also includes the classification of the original data on a scale or according to a typology.

Just a value and a type are sufficient to describe such Statistics.

In a number of cases, however, additional information is needed to describe the Statistics, these cases are covered by specializations of the class Statistics, i.e. the class Quantile, Moment, Discrete Probability and Probability.

The code list of Statistics types is illustrative but gives an idea of what types of Statistics can be described with this application profile.

Statistics
Properties

Property Expected Range Cardinality Description Usage Codelist

Moment

Description
Statistics that indicate how the values are distributed relative to a central measure.
Subclass of
Statistics
Properties
For this entity the following properties are defined: order .
Property Expected Range Cardinality Description Usage Codelist
order Integer 1 Order of the Moment. The mean is considered to be a moment of order 1, variation and standard deviation of order 2, order 3 for the skew of the distribution, and order 4 for the degree to which outlierts widen or narrow the distribution.

Probability

Description
Metric that describes how likely a variable is to exceed a certain threshold (or not).
Subclass of
Statistics
Properties
For this entity the following properties are defined: threshold .
Property Expected Range Cardinality Description Usage Codelist
threshold Threshold 1..* The value that is exceeded. E.g. 10m for the previous e.g.

Quantile

Description
Statistics that relate to a particular quantile.
Subclass of
Statistics
Properties
For this entity the following properties are defined: level .
Property Expected Range Cardinality Description Usage Codelist
level Double 0..1 Number of the quantity to which the Statistics apply. E.g. 1 if the Statistics refer to the first quartile.

StatisticalObservation

Description
Observation whose result is a Statistic.
Subclass of
Observation
Properties
For this entity the following properties are defined: observedCharacteristic , observedObject , Phenomenon time , result , Result time .
Property Expected Range Cardinality Description Usage Codelist
observedCharacteristic Attribute type 1 Characteristic that one observes vh observedObject.
observedObject Object 1 Object observing with.
Phenomenon time TemporalEntity 1 The moment at which the result of the Observation was applicable. Typically the period for which the data on which the Metric is calculated is valid.
result Statistics 1 The value of the observed Characteristic as a result of the Statistical Observation.
Result time Moment 1 Time at which the result of the Observation was obtained. Typically the moment at which the Statistical Observation is calculated.

Statistics

Description
A piece of information that describes a characteristic of a dataset or variable.
Properties
For this entity the following properties are defined: Metric type , value .
Property Expected Range Cardinality Description Usage Codelist
Metric type Metric Type 1 The Statistics type. E.g. average, index etc.
value Any 1 Scope of Statistics. Usually numerically.
