Ce document décrit un profil d’application, dans ce cas Parking pour camions (profil d’application). Ce profil d’application répond à la question de savoir comment le modèle de domaine correspondant peut être appliqué dans la pratique. De plus, les limitations (cardinalité, listes de codes) sont expliquées et les termes correspondants (RDF) sont listés.

Résumé

Le modèle de mise en œuvre d’OSLO Truck Parking montre comment les termes du vocabulaire correspondant doivent être utilisés pour échanger des données sur le stationnement des camions.

Cette norme est issue du projet Smart Truck Parking, avec Truck Parking les initiateurs, dont IGEMO, veulent partager leurs connaissances et leur expérience pour développer un outil numérique qui contribue à la création d’une relation durable entre les chauffeurs routiers et les propriétaires de sites

Le modèle se compose de plusieurs pièces qui ont été réutilisées à partir des normes existantes.

La première partie du modèle traite du camion et de la façon dont il peut se rendre à un parking particulier. Pour clarifier ce processus, des classes telles que Route, Parking et Provider sont utilisées.

Les énumérations sont généralement décrites en dehors du profil d’application, de sorte que les valeurs d’énumération affichées dans le diagramme sont illustratives.
