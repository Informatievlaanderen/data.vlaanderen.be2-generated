Afhankelijkheden
Appplicatieprofiel Generiek-basis

Dit document beschrijft een implementatiemodel, in dit geval Adressenregister. Een implementatiemodel is een toepassing van vocabularia voor een concreet doel. Het beschrijft in alle detail (zoals kardinaliteiten, codelijsten, datatypes, ...) de gewenste datastructuur.

Samenvatting

Dit is een implementatiemodel voor het adressenregister dat het applicatieprofiel Adresregister.

De applicatie waarop dit profiel betrekking heeft is een Vlaamse adresregister. Adresregistratie houdt in dat de beheerders van dit gegeven (in België de gemeenten) adressen en adrescomponenten officieel vaststellen en vastleggen in een register. Het Vlaamse adresregister vormt zo de enige en unieke bron voor adressen in Vlaanderen.

Typisch voor een adresregister is dat de adressen gestructureerd worden. Het adres is een op zichzelf staand object met een eigen identificator en levensloop en bestaat uit adrescomponenten die op hun beurt aan dezelfde kwalificaties voldoen. De adrescomponenten die we hier bedoelen zijn Gemeentenaam, Straatnaam en Postinfo.

Het profiel overstijgt het louter registreren van adressen omdat het bijkomend vastlegt hoe adressen er als attribuut van een ander gegeven moeten uitzien (datatype Adresvoorstelling) of welke bijkomende kenmerken (bv. het volledig adres) er nog van het geregistreerd adres kunnen worden uitgewisseld.

Entiteiten

Adres

Beschrijving
Informatie die toelaat om in België op een gestructureerde en unieke manier te verwijzen naar een "gebouweenheid", "gebouw", een "ligplaats", een "standplaats" of een "perceel" op basis van de naam van de gemeente, de "straatnaam" en het "huisnummer" eventueel aangevuld met een "busnummer" en met een "postcode".
Subklasse van
Object
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: busnummer, heeft gemeentenaam, heeft postinfo, heeft straatnaam, huisnummer, is toegekend aan, is verrijkt met, officieel toegekend, positie, status.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
busnummer String 0..1 Officieel toegekende alfanumerieke code die wordt toegevoegd aan het huisnummer om meerdere gebouweenheden, standplaatsen, ligplaatsen of percelen te onderscheiden die hetzelfde huisnummer hebben.
heeft gemeentenaam Gemeente 1 Gemeentenaamcomponent van het adres.
heeft postinfo Postinfo 1 Postinfocomponent van het adres.
heeft straatnaam Straatnaam 1 Straatnaamcomponent van het adres.
huisnummer String 1 Alfanumerieke code officieel toegekend aan gebouwen, gebouweenheden, ligplaatsen, standplaatsen of percelen.
is toegekend aan Adresseerbaar Object 0..* Adresseerbaar object waaraan het adres is toegekend.
is verrijkt met Adresuitbreiding 0..1 Verwijzing naar een adresuitbreiding.
officieel toegekend Boolean 1 Een adres is officieel toegekend wanneer het werd aangemaakt en/of toegewezen door de gemachtigde adresbeheerder.
positie GeografischePositie 1
status Adresstatus 1 Fase van de levensloop waarin het adres zich bevindt. Link

Adresseerbaar Object

Beschrijving
Geografisch object dat met een adres kan worden geïdentificeerd.
Gebruik
Is abstract, ttz het type adresseerbaar object moet altijd worden opgegeven (vb gebouweenheid, perceel).
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

Adresuitbreiding

Beschrijving
Meer leesbare voorstelling met enkel de basisgegevens van het adres, bedoeld voor het gebruik van een adres als attribuut van een ander object. 
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: volledig adres.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
volledig adres GeografischeNaam 1..* Meer leesbare voorstelling met enkel de basisgegevens van het adres, bedoeld voor het gebruik van een adres als attribuut van een ander object.

Gebouweenheid

Beschrijving
De kleinste eenheid binnen een gebouw die geschikt is voor woon-, bedrijfsmatige, of recreatieve doeleinden en die ontsloten wordt via een eigen afsluitbare toegang vanaf de openbare weg, een erf of een gedeelde circulatieruimte. Een gebouweenheid is in functioneel opzicht zelfstandig. Daarnaast kan een gebouweenheid ook een gemeenschappelijk deel zijn.
Subklasse van
Adresseerbaar Object, Object
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

Gemeente

Beschrijving
Officiële administratieve zone van het Belgisch grondgebied. De gemeente is het kleinste administratieve deel van het Belgisch grondgebied waarvan de grenzen enkel door de wetgever kunnen worden gewijzigd.
Subklasse van
Gemeente
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: faciliteitentaal, gemeentenaam, officiële taal, status.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
faciliteitentaal Taalcode 1..* Ipv. de URIs worden de ISO 639-1 codes {nl, fr, de} gebruikt. Link
gemeentenaam GeografischeNaam 1..* Naam van de gemeente (correcte spelling, eventueel in meerdere talen).
officiële taal Taalcode 1..* Ipv. de URIs worden de ISO 639-1 codes {nl, fr, de} gebruikt. Link
status Gemeentestatus 1 Waarden die de status van een gemeente weergeven. Link

Informatie toegekend door de aanbieder van de universele postdienst voor de identificatie van een groepering van adressen in een geografisch gebied voor postale doeleinden.

Object

Straatnaam

Beschrijving
Naam die officieel werd toegekend ofwel aan een straat (baan, doorgang, plein), ofwel aan een gehucht en waaraan 'huisnummers' kunnen zijn gekoppeld.
Subklasse van
Object
Eigenschappen

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
homoniem toevoeging String 0..1 Toevoeging om dubbele straatnamen (straatnamen met dezelfde naam maar andere ligging in de gemeente en eigen adressen) van elkaar te onderscheiden.
is toegekend door Gemeente 1 Agent die de straatnaam heeft toegekend. In België is dit de gemeente.
status Straatnaamstatus 1 Fase van de levensloop waarin de straatnaam zich bevindt. Link
