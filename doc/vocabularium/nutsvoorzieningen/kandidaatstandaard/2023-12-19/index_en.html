This document describes a vocabularium, for this case OSLO Utility Services. A vocabularium is a collection of reusable terms (classes and properties), whose meaning is given by a label and a definition. There are additional formal agreements available for the terms defined in this domain. The additional formal agreements can be found in external sources for external terms.

Summary

The Utilities vocabulary defines terms that allow to describe a network of utilities.

Status of this document

This section describes the status of this document at the point of publication. It is possible that, in the mean time, newer versions have been released.


A list of the most recent vocabularia can be found here: http://data.vlaanderen.be/ns


This document is an official specification of a vocabulary and provides a shared context of terms for some concepts. This specifcation serves to clarilfy the definition, usage, domain and range for the RDF terms within this domain. The terms defined in this vocabulary are persistent and as a consequence will not change except to further clarify the existing definition. Terms can be marked as deprecated and be replaced in newer versions of this specification.

1. License

This specification of Digitaal Vlaanderen is published under "Model license Free Reuse - v1.0".

2. Conformance

An exchange of data, in whatever way it occurs, conforms to this vocabulary by using the terminology (classes and properties) in a manner that is consistent with the semantics as they are described in the latest version of the specification (domain, range, definition and usage) and does not replace terms in this vocabulary with alternative terms from other vocabularies that cover the same context..

4. Classes

This section defines each class in a formal manner.

Klasse Annotation

Type Class
URI https://data.vlaanderen.be/ns/nutsvoorzieningen#Annotatie
Definition A short explanation or note added to an Object.

Usage

 In this context, typically a Dimension on a cable or pipe map, for example the distance of a cable to a facade. Can consist of parts (e.g. dimensioning line, dimensioning guideline, two arrows) that are also Annotations in themselves.

Klasse Appearance

Type Class

Definition The way an item looks

Klasse Appurtenance

Type Class
URI https://data.vlaanderen.be/ns/nutsvoorzieningen#Toebehoren
Narrower definition for
  • https://data.vlaanderen.be/ns/nutsvoorzieningen#Nutsvoorzieningknoop
Definition Physical point-shaped part of the UtilityNetwork.

Usage

 Can stand alone as a UtilityNode, but is often mounted on a carrier (the UtilityNodeContainer). E.g. an inspection hatch, a hydrant, a splitter, a meter, a pump etc.

Klasse CableAndPipeContainer

Type Class
URI https://data.vlaanderen.be/ns/nutsvoorzieningen#KabelEnLeidingContainer
Definition Object that could potentially contain cables or pipes/ducts.

Usage

 For example, if the object is a Pipe: a Pipe can include Cables or other Pipes. Analogous for pipes of the Duct type. Cables are not CableAndPipe Containers. NOTE: whether the CableAndPipeContainer ultimately contains other elements is indicated by the outgoing relations to those other relations (e.g. Channel.pipe, Pipe.cable, Channel.cable, Pipe.pipe etc).

Definition Collection of linear spatial objects with a specific function or importance in the UtilityNetwork.

Usage

 In practice, a collection of UtilityLinkSequences and/or UtilityLinks. NOTE: is abstract, needs further specialisation e.g. as ElectricityCable, SewerPipe etc.

Klasse UtilityNetwork

Type Class
URI https://data.vlaanderen.be/ns/nutsvoorzieningen#Nutsvoorzieningnetwerk
Narrower definition for
  • https://data.vlaanderen.be/ns/netwerk#Netwerk
Definition Collection of network elements of one type of utility service.

Usage

 For example, an electricity network, a sewerage network, a gas network. The network elements in question are, for example, the elements of the network of an internet provider: optical cables arriving in cabinets on the street via multiplexers connected to copper cables to the homes in that street.
