This document describes a vocabulary, in this case OSLO Data Quality (vocabulary). A vocabulary is a collection of reusable terms, namely classes and properties, whose meaning is recorded through a label and definition. For terms defined in this domain, additional formal agreements are available. For external terms, the formal agreements are copied from the external source.

Summary

The OSLO Data Quality vocabulary is based on ISO19157:2013, a standard for describing data quality.

ISO19157 is intended for geographic data, but is in line with what ISO:25012 (standard for describing the data quality of software products) means by "system-independent data quality".

The vocabulary provides terms to describe the extent to which data is potentially suitable for a particular use. .

Quality element and its subclasses describe various aspects of quality such as accuracy, completeness and consistency and further subdivisions thereof described in ISO1957 (see isotc211:DataQuality for the corresponding uri's). .

In addition to the quality itself, it is possible to specify which quality measure was used and with which method it was calculated. .

The quality can be expressed quantitatively or textually described, or it can be indicated whether the data is compliant or not. .

This specification serves to clarify the definition, use, domain, and scope of the RDF terms within this domain. The terms defined in this vocabulary are persistent and will therefore never disappear, nor will the definitions change except to further clarify an existing definition. 2. Conformity

An exchange of data, by whatever means, is subject to this vocabulary when it uses the terminology (classes and properties) in a manner consistent with the semantics established in the latest version of the specification (domain, scope, definition and usage) and does not use terminology from other vocabularies as an alternative to the proposed terminology included in this vocabulary.

6. External terminology

This section provides an overview of terminology from other vocabularies relevant to this domain model, along with their English language labels and definitions.

accepted

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_ConformanceResult.pass
Definition Indicates whether the result is compliant.

Derivative Element

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_Element.derivedElement
Definition Indicates the original Element, in case of aggregation or derivation.

description

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_DescriptiveResult.statement
Definition Textual description of the Result.

evaluation method

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_Element.evaluationMethod
Definition The evaluation method used.

Evaluation procedure

URI http://def.isotc211.org/iso19157/2013/DataQualityEvaluation#DQ_EvaluationMethod.evaluationProcedure
Definition Reference to the procedure used to evaluate data quality.

evaluationMethodologyDescription

URI http://def.isotc211.org/iso19157/2013/DataQualityEvaluation#DQ_EvaluationMethod.evaluationMethodDescription
Definition Description of the evaluation method.

evaluationMethodType

URI http://def.isotc211.org/iso19157/2013/DataQualityEvaluation#DQ_EvaluationMethod.evaluationMethodType
Definition Type of method used to evaluate data quality.

explanation

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_ConformanceResult.explanation
Definition Explanation of the conformity of the result.

measure

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_Element.measure
Definition Reference to the data quality measure used.

moment

URI http://def.isotc211.org/iso19157/2013/DataQualityEvaluation#DQ_EvaluationMethod.dateTime
Definition Time(s) at which the data quality measure was applied.

moment

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_Result.dateTime
Definition The time at which the result was generated.

ReferenceDoc

URI http://def.isotc211.org/iso19157/2013/DataQualityEvaluation#DQ_EvaluationMethod.referenceDoc
Definition Document(s) referenced in the design and application of data quality evaluation methodology.

result

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_Element.result
Definition Value or values that are the result of the application of a data quality measure, or the result of the evaluation of the value or values obtained based on a given acceptable conformity quality level.

resultScope

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_Result.resultScope
Definition The scope of the result.

sizeDescription

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_MeasureReference.measureDescription
Definition Description of the size used.

sizeIdentification

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_MeasureReference.measureIdentification
Definition Identifier of the size used.

sizeName

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_MeasureReference.nameOfMeasure
Definition Name of the size used.

specification

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_ConformanceResult.specification
Definition Reference to the specification against which the data is evaluated.

standaloneQualityReportDetails

URI http://def.isotc211.org/iso19157/2013/DataQuality_2#DQ_Element.standaloneQualityReportDetails
Definition Free text that shows in detail the methods used, the evaluation, and the result of the data quality.

value

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_QuantitativeResult.value
Definition Quantitative value or values determined by the evaluation method used.

valueRecordType

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_QuantitativeResult.valueRecordType
Definition The type in which the data quality result is reported.

valueUnit

URI http://def.isotc211.org/iso19157/2013/DataQualityResult#DQ_QuantitativeResult.valueUnit
Definition Unit of measure in which a quality result is reported.
