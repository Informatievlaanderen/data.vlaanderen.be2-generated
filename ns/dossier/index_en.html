This document describes a vocabulary, in this case OSLO Dossier. A vocabulary is a collection of reusable terms, namely classes and properties, whose meaning is recorded through a label and definition. For terms defined in this domain, additional formal agreements are available. For external terms, the formal agreements are copied from the external source.

Summary

The conceptual information model "Dossier" forms the basis for the exchange of dossier data between different departments. It defines a number of abstract entities, relevant properties, and their interrelationships. This conceptual information model forms the basis for developers and database managers to further develop into a physical data model. The vocabulary provides an overview of the terminology used and the associated definitions.

This document is an official specification of a vocabulary and provides a shared conceptual framework for certain concepts. This specification serves to clarify the definition, use, domain, and scope of the RDF terms within this domain. The terms defined in this vocabulary are persistent and will therefore never disappear, nor will the definitions change except to further clarify an existing definition. However, terms may be labeled as obsolete and replaced in newer versions of this specification.

This specification of Het Facilitair Bedrijf is published under the "Model License Free Reuse - v1.0".

2. Conformity

An exchange of data, by whatever means, is subject to this vocabulary when it uses the terminology (classes and properties) in a manner consistent with the semantics established in the latest version of the specification (domain, scope, definition and usage) and does not use terminology from other vocabularies as an alternative to the proposed terminology included in this vocabulary.

4. Classes

This section provides a formal definition to each class.

Class Activity context

Type Class
URI https://data.vlaanderen.be/ns/dossier#ActiviteitContext
Specialization of
  • http://www.w3.org/ns/prov#EntityInfluence
Definition Context of a relationship between file documents. This context makes it clear that the documents in the case are consistent in the context of a particular activity.

Class File

Type Class
URI https://data.vlaanderen.be/ns/dossier#Dossier
Specialization of
  • http://www.w3.org/ns/prov#Collection
Definition Coherent set of documents relating to a particular case.

Class Part

Type Class
URI https://data.vlaanderen.be/ns/dossier#Stukonderdeel
Specialization of
  • https://data.vlaanderen.be/ns/dossier#Stuk
  • http://www.w3.org/ns/prov#Entity
Definition A separate part of a larger piece.

Class Piece

Type Class
URI https://data.vlaanderen.be/ns/dossier#Stuk
Specialization of
  • http://www.w3.org/ns/prov#Entity
Definition Unit of evidence in support of a fact.

Class Procedure step

Type Class
URI https://data.vlaanderen.be/ns/dossier#Procedurestap
Specialization of
  • http://dbpedia.org/ontology/UnitOfWork
Definition A procedural step is the entirety of work done within a specific chronological, logical, organizational, or legally determined part of a procedure.

Class Series

Type Class
URI https://data.vlaanderen.be/ns/dossier#Serie
Specialization of
  • http://www.w3.org/ns/prov#Collection
Definition A series is a collection of pieces that shares common traits.
