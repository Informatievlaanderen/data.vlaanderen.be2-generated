This document describes a vocabulary, in this case Vocabulary Financial Reporting. A vocabulary is a collection of reusable terms, namely classes and properties, whose meaning is recorded through a label and definition. For terms defined in this domain, additional formal agreements are available. For external terms, the formal agreements are copied from the external source.

Summary

This vocabulary describes the terminology needed to describe financial reports and some of the data streams in which they are involved.

The following information is captured about the financial report: the financial facts it contains, the financial concept behind the fact (e.g. assets, turnover, costs, usually based on an imposed schedule), the context in which the fact must be viewed (e.g. the period to which the fact relates), if applicable. the dimensions that apply (e.g. time, region, product).

As far as the financial report is concerned, the terminology is based on a well-known standard in this domain, namely XBRL and a sub-standard thereof, namely XBRL-Dimensions.

As far as the data flows are concerned, there are terms to describe the filing of a financial report as well as the various validation steps it goes through, e.g. for approval of a grant. A financial report can also be linked to a file.
